Market Basket Analysis por Segmento

1. Reglas de Asociación Contextuales

El Market Basket Analysis (MBA) clásico asume que todos los clientes tienen las mismas intenciones de compra. Sin embargo, al integrar nuestro modelo de segmentación (K-Means) con las reglas de asociación, logramos descubrir Bundles (paquetes) de productos que son altamente específicos para cada perfil.

Aplicamos el algoritmo de reglas de asociación con un soporte mínimo absoluto, lo que nos permite descubrir que lo que es popular para un “Ocasional” no necesariamente le interesa a un “MVP”.

1.1 Descubrimientos de Canasta por Segmento

Para proteger el catálogo real, los nombres de las familias de productos han sido ofuscados, pero mantenemos las proporciones métricas reales (Soporte, Confianza y Lift) descubiertas en nuestros datos procesados.

Código 
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme(style="white")

# Paleta basada en nuestros segmentos
SEGMENT_COLORS = {
    "MVPs": "#0B3C5D",         # Azul profundo elegante
    "Alto Valor": "#328CC1",   # Azul medio
    "Ocasionales": "#6CA6C1",  # Azul grisáceo suave
    "En Riesgo": "#D82822",    # Naranja rojizo (alerta sin ser agresivo)
    "Hibernando": "#9AA0A6"    # Gris neutro
} 

# Simulación de las reglas de asociación reales por segmento
mock_rules_segment = pd.DataFrame({
    'Segmento': ['MVPs', 'MVPs', 'Alto Valor', 'Ocasionales', 'En Riesgo'],
    'Antecedente': ['Software Corporativo', 'Servidores E.', 'Periféricos C.', 'Accesorio Móvil', 'Consumibles'],
    'Consecuente': ['Licencia Extendida', 'Redes Avanzadas', 'Garantía Estándar', 'Audio Básico', 'Servicio Técnico'],
    'Support (%)': [8.5, 4.2, 12.1, 15.3, 6.4],
    'Confidence (%)': [85.0, 78.5, 62.0, 45.0, 30.0],
    'Lift': [145.2, 120.1, 42.5, 5.8, 2.1]
}).sort_values('Lift', ascending=True)

1.2 Hiper-Asociación en Segmentos Premium

Una de las revelaciones más grandes fue la enorme fuerza de asociación (Lift) en los segmentos VIP. Mientras que los clientes “Ocasionales” compran de manera más caótica (Lifts de ~5x), los “MVPs” tienen compras altamente predecibles y acopladas, alcanzando multiplicadores de propensión de hasta 140x a 150x.

Código 
fig, ax = plt.subplots(figsize=(10, 6))

colors = [SEGMENT_COLORS[seg] for seg in mock_rules_segment['Segmento']]
y_labels = mock_rules_segment['Antecedente'] + " -> " + mock_rules_segment['Consecuente'] + " (" + mock_rules_segment['Segmento'] + ")"

ax.barh(y_labels, mock_rules_segment['Lift'], color=colors, edgecolor="none")

plt.title("Fuerza de Asociación (Lift) Comparativa entre Segmentos")
plt.xlabel("Multiplicador de Probabilidad (Lift)")
plt.axvline(x=1, color='gray', linestyle='--', linewidth=1)
sns.despine()
plt.tight_layout()
plt.show()

2. Accionabilidad Estratégica (Bundles Exclusivos)

En base a esto, hemos generado tablas de Reglas Exclusivas y Reglas con Valor. Esto alimenta directamente a los equipos de Marketing:

  • Campañas B2B para “MVPs”: Los algoritmos detectaron reglas donde las compras de Servidores van acompañadas casi exclusivamente de Licenciamientos Extendidos específicos. Estos bundles no se ofrecen a clientes Ocasionales.
  • Recuperación de “En Riesgo”: A este segmento se le asocian reglas de bajo Lift pero de alto soporte en Consumibles y Servicios. Una campaña ofreciendo un descuento en su consumible habitual es la forma óptima de reactivarlos.
